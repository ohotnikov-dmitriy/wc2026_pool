# World Cup 2026 — Bracket Pool

Турнирная сетка-прогноз для друзей/коллег.

- **Фронтенд** (статические HTML) — на **GitHub Pages**.
- **Бэкенд** (один Worker + база **D1**) — на **Cloudflare** (бесплатно).
- Каждый участник вводит имя+почту → получает **уникальную ссылку**, заполняет сетку
  (группы → 8 лучших третьих мест → плей-офф до финала и матча за 3-е место), сохраняет.
- Правки доступны по ссылке **до дедлайна** (по умолчанию `23:00 CET 10.06.2026`).
  Дедлайн проверяется **на сервере** — после него сохранение отклоняется.
- **Таблица лидеров** считается автоматически по правилам ниже.

## Очки (за верно угаданного участника раунда)

| Раунд | Очки за каждую угаданную команду |
|---|---|
| Round of 32 | 1 |
| Round of 16 | 2 |
| 1/4 (QF) | 3.5 |
| 1/2 (SF) | 5 |
| Матч за 3-е место (участник) | 6 |
| Финал (участник) | 6 |
| **Чемпион мира** | 7.5 |

Максимум за идеальный прогноз — **143.5**. Очко начисляется за каждую команду,
которая в твоём прогнозе дошла до раунда и реально туда дошла (пересечение множеств).

> Примечание: по этим правилам за *победителя* матча за 3-е место отдельных очков нет —
> 6 очков даётся обоим участникам бронзового матча. Если хочешь наградить именно
> бронзового призёра, поменяй формулу в `assets/engine.js` и в блоке `ENG` внутри `worker/worker.js`.

## Структура

```
index.html          вход: имя+почта → создаёт запись → редирект на уникальную ссылку
bracket.html        редактор сетки (?id=...), автодозагрузка, блокировка по дедлайну
leaderboard.html    таблица лидеров
admin.html          ввод официальных результатов (нужен admin-токен)
assets/
  engine.js         данные турнира + подсчёт очков (общий для фронта и Worker)
  bracket.js        логика редактора сетки
  common.js         шапка, флаги, обращения к API
  config.js         <-- ОТРЕДАКТИРУЙ: URL воркера + дедлайн
  app.css, bracket.css
worker/
  worker.js         Cloudflare Worker (API)
  schema.sql        схема D1
wrangler.toml       <-- впиши database_id
```

## Шаг 1. Бэкенд на Cloudflare (~5 минут)

Нужен бесплатный аккаунт Cloudflare и установленный Wrangler (`npm i -g wrangler`).

```bash
wrangler login

# 1) создать базу D1
wrangler d1 create wc2026
#   -> скопируй database_id из вывода в wrangler.toml (поле database_id)

# 2) применить схему
wrangler d1 execute wc2026 --remote --file=worker/schema.sql

# 3) задать секретный admin-токен (придумай любой)
wrangler secret put ADMIN_TOKEN

# 4) (опционально) поправь в wrangler.toml:
#    DEADLINE_ISO  — момент блокировки в UTC
#    ALLOW_ORIGIN  — лучше указать свой домен Pages, напр. https://USERNAME.github.io

# 5) опубликовать
wrangler deploy
#   -> получишь URL вида https://wc2026-pool.<subdomain>.workers.dev
```

## Шаг 2. Фронтенд на GitHub Pages

1. Создай репозиторий и залей в него **все файлы из этой папки** (кроме `worker/` и
   `wrangler.toml` — их можно держать рядом, они не мешают, Pages их просто отдаст как
   текст; при желании вынеси бэкенд в отдельный репозиторий).
2. Отредактируй `assets/config.js`:
   ```js
   API_BASE: "https://wc2026-pool.<subdomain>.workers.dev",  // URL воркера из шага 1
   DEADLINE_ISO: "2026-06-10T21:00:00Z",                      // = 23:00 CET
   DEADLINE_LABEL: "23:00 CET · 10 Jun 2026",
   ```
   Дедлайн в `config.js` — для отображения и мягкой блокировки в браузере; «настоящая»
   блокировка задаётся `DEADLINE_ISO` в `wrangler.toml` на стороне Worker.
3. В настройках репозитория: **Settings → Pages → Source: Deploy from a branch**,
   ветка `main`, папка `/ (root)`. Через минуту сайт будет на
   `https://USERNAME.github.io/REPO/`.

## Шаг 3. Запуск пула

- Раздай людям ссылку на `index.html`. Каждый вводит имя+почту и получает свою ссылку
  `bracket.html?id=...`. Та же почта повторно вернёт ту же ссылку (восстановление).
- По мере турнира заходи на `admin.html`, вводи admin-токен, заполняй **фактические**
  результаты и жми «Save official results». Таблица лидеров обновится сама
  (частичные результаты — норма, недосыгранные раунды дают 0).

## Дедлайн и часовой пояс

`DEADLINE_ISO` хранится в UTC. `23:00 CET` (зимнее время, UTC+1) = `21:00Z`.
Если ты имеешь в виду летнее время **CEST (UTC+2)** — это тоже выходит `21:00Z`
для `23:00`. Если нужен другой момент — поставь нужное `...Z` и в `wrangler.toml`,
и в `assets/config.js`.

## Локальный просмотр

Можно открыть страницы локально (`python3 -m http.server`), но без задеплоенного
Worker запросы к API будут падать — это ожидаемо до шага 1–2.

## Лимиты

Бесплатных тарифов Workers + D1 для пула на десятки/сотни участников хватает с запасом.
Точные текущие цифры лимитов смотри в документации Cloudflare (они периодически меняются).
